# 🐧 Linux & Docker 명령어 정리

## 📁 기본 디렉토리 관리
| 명령어 | 설명 |
|---------|------|
| `pwd` | 현재 경로 확인 |
| `ls -l` | 파일 및 폴더 상세 목록 |
| `cd <경로>` | 디렉토리 이동 |
| `mkdir <폴더명>` | 새 디렉토리 생성 |
| `mkdir -p <a/b/c>` | 중첩 디렉토리 한 번에 생성 |
| `rm <파일>` | 파일 삭제 |
| `rm -rf <폴더>` | 폴더 및 하위 파일 강제 삭제 |
| `mv <대상> <목적지>` | 파일 또는 폴더 이동/이름 변경 |
| `cp <대상> <복사본>` | 파일 복사 |
| `cp -r <폴더> <복사본>` | 폴더 전체 복사 |

---

## 🧾 파일 내용 확인
| 명령어 | 설명 |
|---------|------|
| `cat <파일명>` | 파일 전체 내용 출력 |
| `head -n <숫자> <파일>` | 파일의 앞부분 출력 |
| `tail -n <숫자> <파일>` | 파일의 뒷부분 출력 |
| `grep <문자열> <파일>` | 특정 단어 포함된 줄 검색 |
| `vi <파일명>` | vi 편집기로 열기 (저장: `:wq`, 종료: `:q!`) |
| `echo "문자열" >> <파일>` | 텍스트 추가 저장 |

---

## ⚙️ 파일 권한 관련
| 명령어 | 설명 |
|---------|------|
| `ls -l` | 권한 확인 |
| `chmod 755 <파일>` | 실행 권한 추가 (모두 읽기, 실행 가능) |
| `chmod 700 <파일>` | 소유자만 읽기/쓰기/실행 가능 |
| `sudo -i` | root 권한으로 전환 |
| `exit` | root → 일반 계정 복귀 |

---

## 🧍 사용자 / 그룹
| 명령어 | 설명 |
|---------|------|
| `whoami` | 현재 사용자 확인 |
| `su - <계정명>` | 사용자 전환 (`su - ssafy`) |
| `groups` | 속한 그룹 확인 |
| `sudo usermod -aG docker <계정명>` | docker 그룹 추가 |
| `exit` | 현재 세션 종료 |

---

## 🔍 시스템 & 프로세스
| 명령어 | 설명 |
|---------|------|
| `ps -u $(whoami)` | 현재 사용자 프로세스 보기 |
| `ps aux` | 전체 프로세스 목록 |
| `top` | 실시간 자원 사용량 모니터링 (종료: `q`) |
| `kill <PID>` | 특정 프로세스 종료 |
| `df -h` | 디스크 사용량 확인 |
| `free -h` | 메모리 사용량 확인 |

## 🐳 Docker 기본 명령어

### 🚀 컨테이너 관리
| 명령어 | 설명 |
|---------|------|
| `docker run hello-world` | 도커 정상 작동 테스트 |
| `docker ps` | 실행 중인 컨테이너 목록 |
| `docker ps -a` | 모든 컨테이너 목록 (중지 포함) |
| `docker stop <컨테이너ID>` | 실행 중인 컨테이너 중지 |
| `docker rm <컨테이너ID>` | 중지된 컨테이너 삭제 |

### 🧱 이미지 관리
| 명령어 | 설명 |
|---------|------|
| `docker images` | 보유 중인 이미지 목록 |
| `docker build -t <이미지이름> .` | Dockerfile 기반 이미지 빌드 |
| `docker rmi <이미지ID>` | 이미지 삭제 |
| `docker system prune -f` | 불필요한 캐시/컨테이너 일괄 삭제 |
